# Журнал изменений

Учёт фиксов и улучшений по версиям. Версии компонентов независимы:
`web` (`web/src/version.ts`, `WEB_VERSION`) и ядро `api`/`engine`/`queue` (`__version__`).
Бамп — вручную при заметных изменениях; дата сборки пишется в образ (`/BUILD_TIME`).

Формат: `## <компонент> <версия> — <дата>` и список изменений.
Свежие записи — сверху.

---

## web 1.4.1 — 2026-06-21

- **Широкая раскладка для таблицы.** В табличном виде страница разворачивается почти
  на всю ширину окна (`max-width` контейнера снимается, до 118rem) — колонки больше не
  обрезаются. На карточках и других экранах раскладка прежняя (узкий центрированный
  контейнер).
- **Понятный переключатель вида.** Вместо двух отдельных иконок (вид + плотность) —
  единый сегментированный переключатель **Плитка / Карточки / Таблица**: один клик
  сразу выбирает режим, активный сегмент подсвечен.

## web 1.4.0 — 2026-06-21

- **Табличный вид списка раздач (стиль ruTorrent).** В тулбаре списка — переключатель
  «карточки ↔ таблица» (иконка, выбор сохраняется в `localStorage`, ключ `ui.view`).
  Таблица: плотные строки с зеброй и подсветкой при наведении, липкая шапка, колонки
  Имя / Статус / Размер / Готово (мини-бар + %) / Отдано / Ратио / ↓ / ↑ / Сиды / Пиры /
  Осталось / Метка. Числовые колонки выровнены вправо (моноширинные цифры). Клик по
  заголовку сортирует (использует серверную сортировку: имя/размер/прогресс/отдано/ратио/
  ↓/↑/пиры), активная колонка отмечена индикатором. Чекбокс выбора в каждой строке +
  «выбрать все на странице» в шапке — работает общая панель массовых действий.
  Компактные кнопки в строке: пауза/старт и удаление. Переключатель плотности
  (плитки/строки) относится только к карточкам и в табличном виде скрыт.

## web 1.3.1 — 2026-06-21

- **Чистые URL без `#`.** Роутинг переведён с hash-routing на History API:
  адреса стали `…/torrent/123` и `…/settings` вместо `…/#/torrent/123`. На главной
  больше нет «висящего» `#`. Работают кнопки браузера назад/вперёд, прямые ссылки и
  перезагрузка на глубоком пути (nginx уже отдаёт `index.html` через
  `try_files … /index.html`). Ссылки (`href`) теперь указывают на реальные пути —
  работает средний клик / «открыть в новой вкладке».
